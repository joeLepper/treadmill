- **ADR-0091 quiescence fix — registered task must not pin team non-quiescent (task 690ee089)**: the `open_prs` CTE in `_SNAPSHOT_SQL` (`routers/scheduler.py`) used `derived_status NOT IN ('done', 'cancelled')` to identify "live" tasks behind open PRs. `'registered'` was not excluded, so a task reset to `'registered'` after a prior execution (which left an unclosed `task_prs` row) set `open_pr_with_live_task=True` and made the team appear non-quiescent — blocking the scheduler from pausing it even though no work was in flight. Fix: add `'registered'` to the exclusion set → `NOT IN ('done', 'cancelled', 'registered')`. Module docstring updated to document the exclusion rationale. New test `test_registered_only_task_does_not_pin_quiescence` in `tests/test_scheduler_decision.py` pins the pure-function contract.
