# Plan: Auto-merge on mergeability=mergeable (ADR-0031 execution)

Ship the auto-merge trigger that closes hands-free driving. Phase 1 verifies the five prerequisites are merged before any auto-merge code lands (per ADR-0031 Q31.f, operator framing 2026-05-14).

## Goal

After this plan executes, a PR whose `mergeability` VIEW reads `mergeable` AND whose `wf-validate.decision` is `pass` AND with no pending human review auto-merges via squash after a 30-second cooling-off window, with no operator intervention. Per-plan opt-out (`auto_merge: false` in plan frontmatter) works.

## Success criteria

- `docs/handoffs/2026-05-14-adr-0031-prereq-snapshot.md` exists and confirms all five prereqs landed: #120 (re-author dedup), #121 (author-side validation), #124 (DB reconciliation), ADR-0032's plan (`2026-05-14-adr-0032-documentarian-architect.md`) status=completed, #127 (GH Actions CI).
- `parsers/plan_doc.py` parses `auto_merge: false` from plan frontmatter; persisted on the `Plan` model.
- `coordination/triggers.py` exposes `maybe_auto_merge_on_mergeable`; fires from `coordination/consumer.py` on `mergeability.changed.mergeable` events; respects 30s cooling-off (resettable by intervening events); skips when plan has `auto_merge=false`.
- `dispatch_dedup.py` recognizes the `auto-merge=<task_id>` namespace; emits `task.<id>.auto_merged` events on successful merge.
- End-to-end smoke: a PR opens, transitions through review_passed + validate_passed, mergeability flips to `mergeable`, 30s elapse, auto-merge fires, PR state = MERGED with squash commit on main.

## Constraints / scope

### In scope

- Five tasks below.
- Single-PAT identity (per ADR-0031 Q31.e; separate merger identity awaits #109).
- Per-plan opt-out only (per ADR-0031 Q31.c; not per-task, not per-workflow).

### Out of scope

- Circuit breaker (Q31.d deferred — needs ADR-0098 observability stack for the signal).
- Merge queue (Q31.d rejected for v1).
- Human paging on `uncertain` (Q31.b — `uncertain` routes to wf-feedback for rework, not pages).
- Cross-repo coordination (single-repo for v1).

### Budget

One operator session for plan review + dispatch + smoke. Tasks 2–4 are tractable for Treadmill to execute. Task 1 is operator-mediated (verifies external prereqs); task 5 requires operator to observe the smoke.

## Risks / unknowns

- **Prereq slippage** — if any prereq isn't fully merged at phase-1 verification, the plan blocks. Mitigation: phase 1 produces a snapshot with explicit per-prereq state; an operator can read it and make the merge call (in a future world where one of the 5 has a legitimate carve-out reason to ship later).
- **Merge storms** — a queue of mergeable PRs all flushes at once when the trigger goes live. Mitigation: serialize per-repo via the existing dispatch lock; cap to 1 auto-merge per 10s per repo.
- **Cooling-off races** — events arriving inside the 30s window must reset the timer. Mitigation: store the cooling-off deadline on a Redis key keyed by task_id; consumer extends it on each relevant event. Abort trigger: if smoke shows races losing the merge entirely, extend to 60s and add observability.
