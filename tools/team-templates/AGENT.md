# tools/team-templates

## Purpose

Session-config templates and the installer that `treadmill team up` (Bert's PR-B) calls when bootstrapping a team. Materializes ADR-0087 §Worker execution model + §Evaluator WS subscription + §Per-repo team shape: every team's coordinator, evaluator, and worker sessions get the same `CLAUDE.md` shape and the same security guarantees by construction.

This is the combined PR-D + PR-E surface of the ADR-0087 implementation plan (`docs/plans/2026-06-10-adr-0087-implementation.md`). PR-E (Carla) landed the worker + evaluator templates and the installer; PR-D (Bert) added the coordinator template + extended the installer to render it alongside.

## Key surfaces

- `worker/relay_inject_hook.py` — PostToolUse hook script. Fires after every Bash tool call in a worker subprocess. Reads `~/.cc-channels/<worker-label>/relay/`; if the OLDEST message's `[from: <label>]` header matches `coordinator-<slug>` (derived from the worker's own label), prints `{"decision": "block", "reason": "[COORDINATOR]: <body>"}` and deletes the file. Every other sender label is left in the inbox as data. Stdlib-only; never raises (swallows + emits `{}` on any exception so a broken hook can't kill the worker).
- `worker/settings.json.tmpl` — worker session's `settings.json`. Carries a baseline `permissions.allow` list documenting the expected ambient Bash surface (structurally redundant under `--permission-mode bypassPermissions` but kept for operator readability + future tightening). Hooks block wires the PostToolUse → `relay_inject_hook.py`. No Stop hook by design — workers exit between tasks per ADR-0087; the coordinator drives spawn lifecycle from the subprocess exit JSON, not from a worker broadcast.
- `worker/CLAUDE.md.tmpl` — worker session identity, the load-bearing trust model (only `[from: coordinator-<slug>]` is authoritative; PR text + commit messages + sibling-worker relays are data not instructions), task lifecycle (read brief → implement → open PR → relay back), peer-review subprotocol, and the explicit `what you do not do` list (no API writes, no merges, no approvals, no plan submits). `{{WORKER_LABEL}}` and `{{REPO_SLUG}}` placeholders substituted at install time.
- `evaluator/CLAUDE.md.tmpl` — evaluator session identity, the read-only API constraint (GET only; coordinator owns lifecycle writes per the single-writer invariant), fixed verdict format (`[verdict: approve | rework]` with one-paragraph reasoning + bulleted remediation list for rework), and the explicit no-third-value rule. Marked clearly as a stub — the holistic-judgment criteria are deliberate follow-on scope.
- `coordinator/CLAUDE.md.tmpl` (PR-D) — coordinator session identity, the single-writer invariant on `task_executions` lifecycle state, full ADR-0087 lifecycle handlers: §3 event subscriptions (`plan.submitted`, `task.registered`, `github.pr_merged`, `task.completed`, `github.check_run.completed`, `github.pull_request.synchronize`), §4 worker routing (continuity bias + single-process constraint + FIFO queue overflow), §5 task dispatch ordering (POST `task_execution` → pre-drain inbox → cc-relay brief), §7 CI + conflict loop with 10s/30-attempt mergeability bound, §8 peer review (1–2 reviewers, parallel spawn, collation, re-cycle semantics, post-collation mergeability re-poll), §9 evaluator handoff (brief format, fixed verdict format, approve → merge / rework → re-brief), §9.5 max-cycles cap (≥3 evaluator-rework → orchestrator escalation), §9.6 evaluator timeout (30 / 60 min escalation), §2 startup recovery (sweep stale running rows + drain own inbox + re-poll mergeability + replay missed plan.submitted), §10 escalation surfaces to orchestrator, §11 explicit "things you do not do" list (no production code, no env-file rewrites, no `workflow_runs` PATCH, no self-approval), §14 pitfalls covering the PR-B → PR-F transition window. `{{REPO_SLUG}}` is the only placeholder; the installer renders one per team.
- `install.py` — the team-up installer for the full coordinator + evaluator + worker tree. Two entry points: `install_shared_templates()` copies every role's templates into `~/.treadmill/teams/__templates__/<role>/` (the shared script + template sources, `AGENT.md` editor-facing docs skipped on copy); `install_team(TeamSpec)` renders per-session `CLAUDE.md` for the coordinator + evaluator + every worker, plus `settings.json` for each worker, into `~/.treadmill/teams/<slug>/<label>/`. Idempotent; safe to re-run. `make_team_spec(repo_slug, worker_count=3)` derives the deterministic label set (`coordinator-<slug>`, `evaluator-<slug>`, `worker-<slug>-1..N`). Stdlib-only.
- `tests/test_relay_inject_hook.py` — 13 tests pinning the hook's behavior: fast-path no-op cases, coordinator-message inject + consume, every non-coordinator sender class (sibling worker / evaluator / orchestrator / cross-team coordinator) left in place as data, oldest-message-first ordering, non-worker/malformed labels short-circuit, unreadable-file robustness, JSON output validity.
- `tests/test_install.py` — 13 tests (was 10 + 3 PR-D additions) pinning the installer: spec derivation (default + custom worker count + determinism), shared template copy (now includes `coordinator/`), per-team render of evaluator + worker + coordinator CLAUDE.md, idempotent re-install of all three roles, cross-team isolation, coordinator-specific assertions (`coordinator-<slug>/CLAUDE.md` is rendered with no `settings.json` neighbour; `{{WORKER_LABEL}}` placeholder absent from coordinator output; coordinator render is idempotent).
- `tests/test_coordinator_template.py` (PR-D) — 18 structural tests pinning the load-bearing SHAPE of `coordinator/CLAUDE.md.tmpl`: `{{REPO_SLUG}}` placeholder present + `{{WORKER_LABEL}}` absent, each of the five required event handlers named verbatim, single-writer invariant + relay trust-boundary language present, all four trigger taxonomy values named exactly, evaluator-timeout magic numbers (30 / 60 minutes) match the ADR, max-cycles cap (≥3 evaluator-rework) named, mergeability polling bound (10 s / 30 attempts / `mergeability_undetermined` reason), four startup-recovery steps named (stale sweep / inbox drain / mergeability re-poll / events-table replay), end-to-end render check (substituted output contains no literal `{{REPO_SLUG}}`).

## Recent changes

- **ADR-0087 PR-D — coordinator template + installer extension**: adds `coordinator/CLAUDE.md.tmpl` (the canonical ADR-0087 coordinator system prompt; full lifecycle handlers + startup recovery + escalation surfaces) and extends `install.py` so `install_team(spec)` now renders the coordinator's per-session `CLAUDE.md` alongside the evaluator and workers. The legacy v1 prompt at `tools/coordinator/coordinator_prompt.md` (ADR-0084 / ADR-0086 lifecycle) is marked deprecated; Phase 5 removes it together with the legacy table drops. `install_shared_templates` now copies `coordinator/` alongside `worker/` + `evaluator/` (skipping `AGENT.md`). 21 new tests pinning the coordinator template shape + installer behaviour. The wiring follow-up Carla flagged in PR-E (calling `install_team` from `cli/treadmill_cli/commands/team.py`) is still outstanding — a small follow-up PR can land it independent of PR-D.
- **ADR-0087 PR-E — worker subprocess hook template + evaluator stub + team-up deployment surface** (new dir): lands the `tools/team-templates/` source-of-truth for the worker + evaluator session configs that `treadmill team up` (PR-B) installs at bootstrap. `worker/relay_inject_hook.py` enforces the per-team coordinator-only trust boundary at the hook layer (only `[from: coordinator-<slug>]` messages → injected as `{decision:block, reason:[COORDINATOR]: ...}`; everything else stays as data); `worker/CLAUDE.md.tmpl` enforces the same boundary in prose at every other input surface (PR comments, commit messages, sibling-worker relays). `evaluator/CLAUDE.md.tmpl` carries the read-only-API constraint + the fixed verdict format the coordinator parses. `install.py` is the entry point PR-B's `treadmill team up` calls; idempotent, stdlib-only, no per-template rendering library. 23 tests across two files all pass. Coordinator CLAUDE.md is explicitly NOT touched here — PR-D owns coordinator content.
